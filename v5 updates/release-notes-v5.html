Version 5 — Home Page Redesign & UX Overhaul
Release Date · April 2026 Platform · iOS · Android · Desktop PWA Build · d295faa
1

New Features

New
First-Time Onboarding Modal
New users are greeted with a guided welcome modal explaining the three-step workflow: Scout → Set Bias → Build Your Plan. Includes a payoff summary showing what the app produces (ranked formations, coverage packages, blitz %, call sheets). Dismisses permanently via localStorage flag — never shown again after first visit. Fully accessible with keyboard Escape dismiss and screen reader semantics.
New
X's & O's Hero Section
The home screen now opens with a full-width CSS-rendered X's & O's pattern behind the app title and tagline. No image assets required — pure CSS pseudo-element with multi-directional gradient fades. The left side fades deep to keep the gold title text legible. A matching echo pattern appears below the footer for visual continuity.
New
4+3 Trait Card Grid with Inline Expansion
Scout Traits replaced the previous accordion list with a touch-first card grid: 4 cards on the top row, 3 on the bottom — all stretching edge-to-edge with no orphaned blank cells. Tapping a card expands an inline panel directly below the grid. Only one panel is open at a time. Desktop widens to 7 cards across in a single row.
New
Numbered Section Anchors
The home page now has clear visual waypoints — ① Scout Traits, ② Run / Pass Bias, ③ Build — with gold circle numbers and gold-to-transparent divider lines. Users can orient themselves at a glance without needing to scroll to discover what's next.
New
Slim Sticky Top Bar
A compact sticky header replaced the old large header. It shows the CFB · Defensive Intelligence sub-brand on the left and the playbook selector button on the right. Handles iOS Dynamic Island / notch safe area automatically via env(safe-area-inset-top).
2

Visual Updates

Visual
Across-the-Board Text Size Increases
Phone readability improved throughout: trait card labels 8px → 10px, section anchor labels 9px → 11px, top bar sub-brand 8px → 11px, hero tagline 12px → 14px, bias slider labels 11px → 13px, and build section stats bumped accordingly. Card emoji icons increased 15px → 17px.
Visual
XO Pattern Brightness & Fill
Pattern opacity increased from 10% to 35% for better visibility on mobile screens. Each pattern row extended to 24+ X/O pairs, filling the full 720px container width edge-to-edge on all screen sizes. Hero area padding increased to give the background pattern more visible space above and below the title.
Visual
Gender-Neutral Labels
"His Personnel" renamed to "Personnel" across the trait card and expansion panel — inclusive language for all coaching contexts.
3

Bug Fixes & Quality

Fix
Onboarding Modal — React Portal
The modal overlay is rendered via createPortal into document.body, preventing the CSS entry animation's transform from clipping the fixed overlay to the 720px container instead of the full viewport.
Fix
iOS Safe Area Inset
The playbook dropdown position accounts for env(safe-area-inset-top) so it appears correctly below the sticky bar on iPhones with a Dynamic Island or notch.
Fix
Trait Panel Null Guard
Added defensive null check in the inline expansion panel's TRAITS.find() call to prevent crashes if openCard state references a non-existent group ID.
